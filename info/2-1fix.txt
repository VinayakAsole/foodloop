📋 Phase 2: Refined Functional Requirements1. Seller (Home Cook)Register & complete profile (Awaiting Admin Verification).Post available food listings with quantity, price, expiry timestamp, and photo.Live Kitchen Status Toggle: Switch instantly between 🟢 Ready, 🟡 Cooking, or 🔴 Sold Out.Donation Toggle: Convert remaining unsold food to free donation with 1 click when expiry approaches.Verify order pickups by entering a secure Buyer OTP.2. Buyer (Student / Local Resident)Browse nearby available meals sorted strictly by distance (calculated via the Haversine Formula).View instant, non-refreshing Live Food Counters (e.g., "3 plates left").Place orders and receive a secure Pickup OTP.Scan or upload a meal image to identify the item automatically (TensorFlow.js).3. Admin (Platform Quality Control)Review, verify, or suspend Seller accounts.Monitor global platform statistics (Total meals saved, total donations processed).🗄️ Phase 3: Patched Database Schema (MySQL)These schemas contain the newly added fields required for verification, security, and the donation tracking logic.1. users TableSQLCREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    mobile VARCHAR(15) NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('buyer', 'seller', 'admin') DEFAULT 'buyer',
    status ENUM('pending', 'verified', 'suspended') DEFAULT 'pending', -- FIX: Added for Admin verification
    latitude DECIMAL(10, 8) NOT NULL,
    longitude DECIMAL(11, 8) NOT NULL,
    profile_photo VARCHAR(255) NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
2. foods TableSQLCREATE TABLE foods (
    id INT AUTO_INCREMENT PRIMARY KEY,
    seller_id INT NOT NULL,
    food_name VARCHAR(255) NOT NULL,
    category VARCHAR(100) NOT NULL,
    description TEXT NULL,
    price DECIMAL(8, 2) DEFAULT 0.00,
    quantity INT NOT NULL,
    remaining_quantity INT NOT NULL,
    expiry_time TIMESTAMP NOT NULL,
    image_url VARCHAR(255) NULL,
    status ENUM('available', 'sold_out', 'expired') DEFAULT 'available',
    is_donation BOOLEAN DEFAULT FALSE, -- FIX: Added for social impact tracking
    FOREIGN KEY (seller_id) REFERENCES users(id) ON DELETE CASCADE
);
3. orders TableSQLCREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    buyer_id INT NOT NULL,
    seller_id INT NOT NULL,
    food_id INT NOT NULL,
    quantity INT NOT NULL,
    total_price DECIMAL(8, 2) NOT NULL,
    payment_status ENUM('pending', 'completed') DEFAULT 'pending',
    order_status ENUM('placed', 'ready_for_pickup', 'completed', 'cancelled') DEFAULT 'placed',
    otp_code VARCHAR(6) NOT NULL, -- FIX: Secure 6-digit verification code for secure handoff
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (buyer_id) REFERENCES users(id),
    FOREIGN KEY (seller_id) REFERENCES users(id),
    FOREIGN KEY (food_id) REFERENCES foods(id)
);
🤖 Phase 4: Simplified AI Features (Viva-Proof)Instead of complex backend data science models that require immense computational resources, use client-side or rule-based AI logic:AI Food Recognition (Client-Side ML): Train a custom model using Google’s free Teachable Machine. Export it as a lightweight model.json bundle and load it directly inside your React app using @tensorflow/tfjs. This runs completely free on the user's browser engine.Smart Price Suggestion: When a seller types in an item name (e.g., "Poha"), Laravel runs a fast aggregate query checking the average price of that category within a 5km radius and suggests the optimal rate.Data-Driven Demand Analytics: Instead of deep learning predictions, use native SQL grouping on your orders table. Display a lightweight analytics card to the seller: "75% of breakfast items in your area sell out between 8:00 AM and 10:30 AM. Consider adjusting your cooking schedule."
